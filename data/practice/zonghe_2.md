1、 *[单选]* 关于控制质量工具的描述，不正确的是 ( ) 。

- A：核查表用于合理排列各种事项，以便有效地收集关于潜在质量问题的有用数据
- B：因果图用于识别质量缺陷和错误可能造成的结果
- C：核对单用于合理排列各种事项，以便有效地收集关于潜在质量问题的有用数据
- D：控制图用于确定一个过程是否稳定，或者是否具有可预测的绩效

正确答案：**C** 你的答案：**A**

解析：①考点提炼：本题主要考察质量管理工具的区别与应用场景。  
②正确答案解析：C选项错误地将核对单（Checklist）描述为“用于合理排列各种事项，以便有效地收集关于潜在质量问题的有用数据”。  
③其他答案不正确解析：A选项：核查表（Check Sheet）通过表格形式分类记录数据，便于收集和分析质量问题发生的频率、类型等，描述准确。B选项：因果图（鱼骨图）用于追溯问题根源，识别导致质量缺陷的潜在原因（如人、机、料、法、环等），而非“可能造成的结果”。但命题中“识别质量缺陷和错误可能造成的结果”可理解为“分析缺陷产生的可能原因”，在PMBOK语境中因果图的核心是归因分析，此描述可接受。D选项：控制图（Control Chart）通过设置控制界限（上下限）监控过程是否稳定、是否受控，描述完全准确。  
④考点延展：Check Sheet（核查表）和Checklist（核对单）常被混淆。其他质量工具如帕累托图（识别主要问题）、散点图（分析变量关系）、直方图（数据分布分析）也需掌握其区别。

2、 *[单选]* 关于数据安全的说法，错误的是 ( ) 。

- A：数据安全策略属于组织的数据安全能力域的能力项
- B：在制定数据安全策略的过程中需要结合组织管理需求、监管需求以及相关标准等统一制定
- C：数据安全策略是数据安全的核心内容
- D：数据安全审计的目标制定防范措施并监督落实

正确答案：**D** 你的答案：**D**

解析：①考点提炼：本题主要考察 《数据安全法》及数据安全管理体系中关于数据安全策略和数据安全审计的核心概念与定位。  
②正确答案解析：D选项错误地描述了数据安全审计的目标。  
③其他答案不正确解析：A选项：在主流数据安全框架（如DSMM数据安全能力成熟度模型）中，“数据安全策略”明确属于“组织建设”或“治理域”下的一个关键能力项，是构建安全能力的基础。B选项制定数据安全策略必须进行多维度需求整合，包括：组织内部的管理与业务需求、外部法律法规和监管要求（如《数据安全法》）、以及国内外相关标准（如ISO 27001、GB/T 37988）。描述准确。C选项：数据安全策略是数据安全管理的顶层设计和指导性文件，为所有安全活动提供原则、规范和方向，因此确实是整个数据安全工作的核心内容。描述无误。  
④考点延展：策略 (Policy) vs. 审计 (Audit)：策略是“事前规定”，审计是“事后检查”。审计目标：牢记“评估、验证、建议”六字核心，而非直接“制定措施”或“负责落实”

3、 *[单选]* 以下关于项目可行性研究的说法中，错误的是 ( ) 。

- A：经济可行性分析主要是对整个项目的投资及所产生的经济效益进行分析
- B：辅助 (功能) 研究只能作为初步可行性研究、详细可行性研究和大规模投资建议的前提或辅助
- C：详细可行性研究报告视项目的规模和性质，有简有繁
- D：初步可行性分析是必不可少的

正确答案：**D** 你的答案：**D**

解析：①考点提炼：本题主要考察 项目可行性研究的核心概念、阶段划分和作用，  
②正确答案解析：D选项初步可行性研究（Pre-feasibility Study）并非所有项目的强制性或“必不可少”的步骤。  
③其他答案不正确解析：A选项：经济可行性分析（Economic Feasibility Analysis）的核心内容确实是评估项目的总投资成本（如建设投资、流动资金）和预期经济效益（如营业收入、利润、投资回收期、净现值NPV、内部收益率IRR等），以判断项目在经济上是否合理、是否具有财务可持续性。描述准确。B选项：辅助（功能）研究（Auxiliary/Functional Study）确实是针对项目的关键环节或特定方面（如市场需求、原材料供应、技术试验、厂址选择等）进行的专题研究。这些研究的结论主要是为初步或详细可行性研究提供深入、可靠的数据和支持，作为其前提或辅助依据。描述符合其核心定位。C选项：详细可行性研究报告（Detailed Feasibility Study Report）的深度、广度和详细程度，确实取决于项目的规模（投资额大小）和性质（如新建工厂、IT系统开发、基础设施等）。大型复杂项目报告可能长达数百页，而小型项目可能相对简化。描述符合实际。

4、 *[单选]* 在敏捷型或适应型项目生命周期中，( ) 通过迭代对工作进行指导和管理。

- A：规划过程组
- B：执行过程组
- C：监控过程组
- D：收尾过程组

正确答案：**B** 你的答案：**B**

解析：①考点提炼：本题考察过程组与项目生命周期的结合应用，  
②正确答案解析：B选项在敏捷或适应型生命周期中，项目工作通过一系列短周期迭代来推进。执行过程组的核心作用是完成项目管理计划中确定的工作，以满足项目规范要求。  
③其他答案不正确解析：A选项：在敏捷中，规划是持续进行的（如迭代计划会），但它是为执行服务的。敏捷强调“刚好足够的规划”，而不是一次性、详尽的预先规划。规划活动被嵌入到每个迭代中，作为执行的前置步骤，但其本身不是对迭代工作进行持续指导和管理的主导过程。C选项：监控过程组（跟踪、审查、调整）在敏捷中至关重要（如每日站会、迭代评审会），它确保执行不偏离目标，并触发调整。但监控主要是跟踪和评估工作，其核心职能是“检查”，而不是直接“指导和管理”工作的开展。指导和管理是执行过程组的核心职能。D选项：收尾过程组主要发生在迭代结束（发布）或项目结束时，用于正式结束迭代或项目。它不具有在迭代过程中进行持续指导和管理工作的特性。  
④考点延展：指导与管理项目工作（Direct and Manage Project Work） 是执行过程组的一个关键过程，在敏捷中表现为管理迭代过程、协调团队、创造价值交付物。监控项目工作（Monitor and Control Project Work） 是监控过程组的过程，侧重于跟踪、审查、报告绩效。

5、 *[单选]* 下列关于资源优化技术的描述错误的是 ( ) 。

- A：资源平衡往往导致关键路径改变
- B：如一个资源在同一时段内被分配至两个或多个活动，就需要进行资源平衡
- C：资源平滑技术可以实现所有资源的优化
- D：资源平滑技术不会改变项目关键路径，完工日期也不会延迟

正确答案：**C** 你的答案：**C**

解析：①考点提炼：本题主要考察项目管理中资源优化技术的核心概念与应用。  
②正确答案解析：资源平滑是一种受约束的优化，而资源平衡则可能通过延长工期来实现资源的根本性均衡。因此，C选项夸大了资源平滑的能力。  
③其他答案不正确解析：A选项：资源平衡旨在解决资源过度分配（如资源冲突）或资源使用量大幅波动的问题。其典型做法是将非关键路径上的活动在其时差范围内延迟。当非关键活动的时差被用完，或者为了平衡资源而必须延迟关键活动时，关键路径就很可能被延长，从而导致项目总工期增加和关键路径改变。这是资源平衡的典型特征。B选项：这是资源平衡的典型触发场景。当同一个资源（例如一位工程师）在同一时间段内被分配给两个或多个需要全职工作的活动时，就产生了资源过度分配。为了解决这种不可行的冲突，必须进行资源平衡，通过调整活动的开始和结束时间来确保资源在任何时间点只承担合理的工作量。D选项：这准确地描述了资源平滑技术的核心特征。资源平滑是在浮动时间（时差）内调整非关键活动，以确保资源使用量不超过预定的上限，并尽可能平稳。由于它不调整关键活动，因此不会改变关键路径，也不会导致项目总工期（完工日期）延迟。  
④考点延展：“平（资源平）衡改关键，工期会延长；平滑不动窝（关键路径），只梳非关键。”

6、 *[单选]* 关于领导者和管理者的描述，不正确的是 ( ) 。

- A：领导者的主要工作是确定方向、统一思想、激励和鼓舞
- B：管理者主要关心持续不断地为干系人创造他们所期望的成果
- C：领导者设定目标，管理者率众实现目标
- D：对于大型复杂项目，管理能力尤为重要

正确答案：**D** 你的答案：**B**

解析：①考点提炼：本题考察项目环境下“领导者”与“管理者”角色定位的核心区别。  
②正确答案解析：D选项“尤为重要”的片面性：大型复杂项目的特点是不确定性高、干系人众多、目标动态变化、对创新和变革需求大。面对这种环境，领导能力（如建立愿景、引领变革、激励团队、化解冲突）与管理能力（如计划、组织、控制）同等重要，甚至更为关键。只强调管理能力“尤为重要”是片面的。  
③其他答案不正确解析：A选项：这精准概括了“领导者”（Leader）的核心职能。“确定方向”指设定愿景和战略；“统一思想”指凝聚共识、对齐目标；“激励和鼓舞”指激发团队成员的内在动力和承诺。这些都是领导力的典型体现。B选项：这描述了“管理者”（Manager）的核心职责。管理者侧重于计划、组织、指挥、协调、控制，通过建立和维护流程、系统来确保项目按既定轨道运行，从而持续稳定地交付成果，满足干系人期望。这是管理工作的本质。C选项：这是对领导者与管理者角色分工的经典描述。领导者（如项目发起人、高层领导）负责设定目标和描绘愿景；管理者（如项目经理）则负责组织团队、制定计划、领导团队执行，将目标转化为现实。“率众实现目标”正是管理者的核心使命。  
④考点延展：在现代项目管理中，项目经理必须身兼领导者和管理者双重角色。

7、 *[单选]* ( ) 不属于概率估算的相关因素。

- A：具有一定区间的点估算
- B：置信程度
- C：概率分布
- D：相对估算

正确答案：**D** 你的答案：**B**

解析：①考点提炼：本题考察项目估算技术中“概率估算”的核心概念。  
②正确答案解析：D选项“相对估算”不属于概率估算的相关因素，相对估算是敏捷等方法中常用的一种估算技术，但它本身不涉及概率、置信区间或概率分布。因此，它不是构成概率估算模型的相关因素。  
③其他答案不正确解析：A选项：这是概率估算的核心输入或表现形式之一。概率估算通常从一个基准的点估算（如最可能值）开始，然后考虑风险与不确定性，将其扩展为一个可能值的区间（如三角分布的乐观值、最可能值、悲观值）。因此，该描述是概率估算的典型特征。B选项：置信程度（Confidence Level，如90%的置信度）是概率估算的核心输出要素。它表示实际结果落在某个估算区间内的可能性。例如，“我们有90%的信心认为项目成本在100万至150万之间”。没有置信程度的估算不能称为完整的概率估算。C选项：概率分布（如三角分布、正态分布、贝塔分布）是概率估算的数学模型基础。它描述了各种可能结果及其对应的发生概率。在进行蒙特卡洛模拟等概率分析时，必须为每个活动或成本项指定一个概率分布，以模拟总体的不确定性。  
④考点延展：关注不同类型估算方法的对比：确定性估算（点估算）、相对估算、三点估算（PERT）、概率估算。

10、 *[单选]* ( ) 属于物理配置审计。

- A：配置项的开发已圆满完成
- B：配置项已达到配置标识中规定的性能和功能特征
- C：配置项的操作和支持文档已完成并且是符合要求的
- D：要交付的配置项是否存在

正确答案：**D** 你的答案：**D**

解析：①考点提炼：本题考察配置审计的类型与具体内容。  
②正确答案解析：D选项“要交付的配置项是否存在”属于物理配置审计的典型内容，“要交付的配置项是否存在”是对物理实体存在性的最基本验证，是物理配置审计的首要和基础任务。  
③其他答案不正确解析：A选项:这是功能配置审计的内容，而非物理配置审计。功能配置审计关注的是过程与性能，它验证配置项的开发是否按照需求、设计和规范正确完成，其功能是否符合预期。它检查的是“事情做得对不对”，而非“东西在不在”。B选项：这同样是功能配置审计的核心内容。它通过测试、评审等方式，验证配置项的实际性能、功能和行为是否达到了需求规格说明书和设计文档中规定的要求。C选项：这属于物理配置审计的范畴吗？不完全是，且更偏向功能审计的延伸。虽然物理配置审计会检查文档的“存在性”和“完整性”，但“是否符合要求”通常涉及对文档内容准确性、与实物一致性的深度审查，这往往是功能配置审计的一部分，或者是两者结合的审计点。因此，它不是一个纯粹的物理配置审计的典型描述，且不如D选项直接和准确。  
④考点延展：记忆技巧与易错点：物理 = 实体存在：记住“物理”二字，联想到实实在在的“物体”和它的“纸质/电子文档”是否存在、是否齐全。功能 = 行为性能：记住“功能”二字，联想到它是否能“工作”、工作得“好不好”。

- A：确保所有变更请求都得到及时处理
- B：对实物资源进行了有效管理

34、 *[单选]* 关于项目管理计划的描述，不正确的是 ( ) 。

- A：在项目收尾之前，项目管理计划需要通过不断更新来渐进明细，并且这些更新需要得到控制和批准
- B：项目管理计划可以是概括或详细的

②正确答案解析：C选项“在确定基准之前，对项目管理计划更新也需遵循正式的变更流程”的描述是不正确的。在项目管理计划被批准并确定为基准之前，对其内容的修改通常属于规划过程本身的一部分，是正常的“完善”和“更新”。这些更新由项目经理和团队主导，可能只需在团队内部或与关键干系人进行非正式的评审和确认，而不一定需要走完整的、正式的变更控制流程。

35、 *[单选]* ( ) 开发方法相对稳定，范围、进度、成本、资源和风险可以在项目生命周期的早期阶段进行明确定义。

- A：预测型方法
- B：迭代型方法
- C：增量型方法
- D：敏捷方法

正确答案：**A** 你的答案：**A**

解析：①考点提炼：本题考察不同项目生命周期（开发方法）的核心特征和适用场景。  
②正确答案解析：A选项 “预测型方法”（又称瀑布型方法）最符合题干描述。题干中的关键词“相对稳定”和“可以在项目生命周期的早期阶段进行明确定义”，精准地概括了预测型方法的根本特征。  
③其他答案不正确解析：B选项：迭代型方法通过一系列重复的循环（迭代）来逐步完善产品。在每个迭代开始时，范围和目标可能会被调整和细化。因此，虽然项目早期可能有一个高层次的方向，但详细的范围、成本等并不是在项目早期就完全“明确定义”并“稳定”不变的，它们会随着迭代和反馈而演进。C选项：增量型方法通过一系列按时间顺序排列的增量来逐步交付完整的产品功能。每个增量都增加一部分功能。虽然每个增量的范围可以提前定义，但整个项目的最终范围可能在早期并不完全明确，或者可以根据早期增量的反馈进行调整。同样，它不具备题干所强调的“早期全面明确定义并保持稳定”的特点。D选项：敏捷方法是适应型和迭代增量型的结合，其核心就是拥抱变化。它明确承认在项目早期无法明确定义所有需求，范围、进度甚至成本都可能随着项目的推进和干系人反馈而动态调整和演进。因此，它与题干描述的“早期明确定义且相对稳定”的特征完全相反。  
④考点延展：迭代与增量：迭代着重于完善，每一次循环都使产品更接近最终目标（可能每次循环都处理产品的所有方面）。敏捷通常是迭代和增量的结合，即通过短周期迭代，每次迭代都交付一个产品增量。

- C：丙

75、 *[单选]* ( ) 显性知识的主要特征不包括。

- A：客观存在性
- B：实践性
- C：可共享性
- D：认知元能性

正确答案：**B** 你的答案：**B**
